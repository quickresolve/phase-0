Git and Github

January 2, 2016

An essential concept for developers is version control. Version control is a relatively simple concept focused on a single concept of tracking changes to a file. There benifits to version control are numerous:

  1. Reverting back to a previous version
    There are many instances where one might want to revert back to a previous version. If a mistake is made with the code, it might be easier to revert to an older version. This also encourages experimentation with code. Changes can be made to a file with no ramifications to the project.
  2. Allowing others to work on the same file
    Utilizing version control can help with multiple people working on the same project. When a new version is added, the developer working on it can add a note about what changes were added.
  3. Tracking progress of a project
    Tracking progress can help for a few reasons. First it can help with accountability in that it can show who made what changes and when. If the project is open sourced, it can also provide more visability to the community it is serving.
